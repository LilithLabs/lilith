# Schedule -- Proactivity

Help the user set up work that runs on its own.

1. Confirm WHAT should run (e.g., "a morning brief", "a weekly spending check"),
   WHEN (cadence or a one-time future moment), and which project it concerns.
2. Create the scheduled task using the host's scheduling capability.
3. Confirm back in plain language: what will run and when. Offer to change or
   cancel it later.

Good defaults to offer during onboarding: a daily brief, and a weekly check on
the user's first project (e.g., new transactions for Finance).
